# Punch — Android app

A [Capacitor](https://capacitorjs.com) wrapper around the Punch web frontend, producing an installable Android `.apk`. The app loads its UI from the live VPS (`server.url` in `capacitor.config.json`), so **most updates need only a frontend redeploy, not a new APK**.

See the [top-level README](../README.md) for the full system overview.

## How it works

- `capacitor.config.json` sets `server.url = https://152-42-157-132.sslip.io` → the WebView loads the app from the VPS.
- The bundled `www/` (a copy of the frontend build) is a fallback; the native project glue under `android/` is generated by `npx cap sync`.
- The APK is **built in the cloud via GitHub Actions**, not locally (local Windows builds were blocked by an AF_UNIX/Gradle-daemon issue on the dev machine).

## Build the APK (GitHub Actions)

1. Push this `android-app/` folder to a GitHub repo:
   ```bash
   git remote add origin https://github.com/<you>/punch-android.git
   git push -u origin main
   ```
2. The workflow `.github/workflows/build-apk.yml` runs automatically:
   `npm ci` → `npx cap sync android` → `gradlew assembleDebug`.
3. Open the repo's **Actions** tab → the green run → **Artifacts** → download **`punch-debug-apk`**.
4. Unzip → `app-debug.apk` → transfer to the phone → install (allow "install from unknown sources").

The workflow can also be triggered manually from the Actions tab (`workflow_dispatch`).

### Why each CI step exists

| Step | Reason |
|---|---|
| Node **22** | Capacitor CLI requires Node ≥ 22 |
| `npm ci` | `android/capacitor.settings.gradle` references `node_modules/@capacitor/android` |
| `npx cap sync android` | Regenerates gitignored native glue (cordova plugins, web assets, capacitor config) |
| JDK **21** | The current Capacitor Android library compiles for Java 21 |

## Configuration

- **App id / name**: `co.punch.app` / "Punch" (`capacitor.config.json`)
- **Icons / splash**: generated from `assets/` via `@capacitor/assets` into `android/app/src/main/res/`
- **Backend**: entered at runtime in the app's Settings screen (nothing sensitive is baked into the APK)

## Updating

- **UI / cosmetic changes** → rebuild & redeploy the **frontend** to the VPS, then reopen the app. No APK rebuild.
- **App id, icon, name, native plugins, or `server.url`** → change here, commit, push → CI builds a new APK to reinstall.

## Local build (only if the dev machine's Gradle/AF_UNIX issue is resolved)

```bash
npm ci
npx cap sync android
cd android
# JAVA_HOME must point to a JDK 21; ANDROID_SDK_ROOT to an Android SDK
./gradlew assembleDebug
# → app/build/outputs/apk/debug/app-debug.apk
```

## Trade-off of `server.url`

The app needs to reach the VPS at launch to load its UI (the interface is no longer fully offline-bundled). For a trading tool that's useless without the backend anyway this is fine, but if the VPS is down the app shows a blank screen. To bundle the UI offline instead, remove the `server` block from `capacitor.config.json` and rebuild.
